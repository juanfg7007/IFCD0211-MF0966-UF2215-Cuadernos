UF2215. UD3. Actividad 4. Caso práctico: Herramientas CASE - Eclipse


Enunciados (Descripción)

Vamos a trabajar ahora con una herramienta CASE llamada Eclipse.

Para ello lo primero que tenemos que hacer es descargarnos el archivo ZIP para la versión Windows de 32 bits llamada “eclipse SDK 4.2.1 win32”.

Lo podemos hacer desde este enlace;

http://archive.eclipse.org/eclipse/downloads/drops4/R-4.2.1-201209141800/

Eclipse es un sistema de control de versiones que mantiene un registro de todo el trabajo y todos los cambios en un conjunto de archivos, y que permite que varios desarrolladores puedan colaborar adecuadamente a la hora de realizar modificaciones sobre los mismos.


Preguntas / Actividades a realizar 

Pregunta 1. Compara dos archivos escritos en código Java mediante la herramienta Eclipse. Para ello crearemos un proyecto llamado Prueba donde crearemos dos clases:

En la primera clase o archivo Java llamado Comparar1 tendremos el siguiente código:

publicclass Comparar1 {
/**
* @paramargs
*/
publicstaticvoid main(String[] args) {
String hi="Hola";
System.out.println(hi);
}
}


En el otro archivo Java que llamaremos Comparar2 tendremos este código:

publicclass Comparar2 {
/**
* @paramargs
*/
publicstaticvoid main(String[] args) {
System.out.println("Hola");
}
}


Muestra los cambios que existen entre ambos archivos al compararlos mediante la herramienta CASE Eclipse.

UD3 - Actividad 4 - Resolución

Pulsa aquí para visualizar: UD3 - Actividad 4 - Resolución

Los cambios son:

  • Comparar1.java: Declaración y asignación de la variable 'hi'.
  • Comparar2.java: Eliminación de la variable `hi` y directamente imprime "Hola" en la línea correspondiente.

Estos cambios indican que en el archivo Comparar1.java se usa una variable adicional 'hi' para almacenar la cadena "Hola", mientras que en el Comparar2.java, la cadena "Hola" se imprime directamente sin utilizar una variable intermedia.
