UF2215. UD1. Actividad 2. Caso práctico: GBD MySQL


Enunciado (Descripción)

Para la realización de esta actividad debes de crear una Base de Datos Documental en Knosys que se llame “LIBROS.DOK” para almacenar libros.


Preguntas / Actividades a realizar 

Pregunta 1. En este ejercicio vamos a hacer uso del SGBD MySQL. Para ello debemos realizar previamente una instalación de este gestor de base de datos.

Pregunta 1A. Haz una instalación predeterminada del SGBD MySQL. Para ello lo primero que tienes que hacer es bajarte una versión de MySQL. Por ejemplo, la versión para Windows mysql-5.5.37-win32.msi de esta dirección:
Fuente: http://dev.mysql.com/downloads/file.php?id=451319

En este proceso indicaremos qué elementos queremos instalar. Por defecto instalaremos todo menos los “DevelopersComponents“, que son opcionales, y contienen ejemplos, scripts y algunas librerías.

Pregunta 1B. Dentro de la consola de MySQL realiza las siguientes operaciones:

  • Muestra las bases de datos existentes en el sistema.
  • Crea una base de datos y llámala“miprueba”.
  • Usa esta tabla para crear una tabla llamada prueba como la siguiente:
    prueba (id_prueba int);
  • Inserta un valor en esta nueva tabla creada.
  • Obtén información de la tabla nueva creada.
  • Selecciona el campo de dicha tabla.

Pregunta 1C. Dentro de la consola de MySQL realiza las siguientes operaciones:

  • Analiza y almacena la distribución de claves en la tabla prueba creada anteriormente.
  • Reorganiza y compacta los datos así como reconstruye el archivo de índices reduciendo el número de accesos a disco necesarios para leer la tabla prueba optimizando su funcionamiento.
  • ¿Cuál es el tipo de motor en MySQL más adecuado para aplicaciones con poca concurrencia y muchos accesos de lectura?

Pregunta 1D. Dentro de la consola de MySQL realiza las siguientes operaciones:

  • Crear un usuario llamado alberto con contraseña “abcdef”
  • Asignar todos los privilegios posibles al usuario “alberto” (si no existe se crea) sobre todas las bases de datos y tablas, para conexiones desde localhost únicamente y además concédele el permiso GRANT para que tenga la opción de asignar permisos a otros usuarios.

El tipo de motor más adecuado para aplicaciones con poca concurrencia y muchos accesos de lectura es el motor MyISAM. Este motor de almacenamiento es eficiente para operaciones de lectura y adecuado cuando no hay muchas operaciones de escritura concurrentes. Sin embargo, no es transaccional y no soporta bloqueos a nivel de fila, lo que puede afectar en entornos con alta concurrencia de escritura.
