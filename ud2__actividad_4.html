UF2215. UD2. Actividad 4. Caso práctico: Migración


Preguntas / Actividades a realizar 

Pregunta 1. Describe los pasos para realizar una migración de una base de datos Access a una base de datos MySQL, usando la herramienta de prueba ESF Database Migration Toolkit Edición Professional e interpretando la documentación inherente a esta herramienta específica.

Para ello puedes hacer uso, si quieres, de la base de datos Access llamada "Videoclub", que podemos encontrar en el siguiente enlace:

http://www.teformas.com/2012/12/ejercicio-2-access-crear-relacionar-tablas.html

En el hipervínculo Solución podemos descargarnos este ejemplo de base de datos Access para poderlo usar en este ejercicio.

UD2 - Actividad 4 - Resolución

Pulsa aquí para visualizar: UD2 - Actividad 4 - Resolución

Pasos para migrar una base de datos de Microsoft Access a MySQL utilizando la herramienta ESF Database Migration Toolkit Edición Professional:

  • Descargar e instalar ESF Database Migration Toolkit:
    • En el sitio web oficial de ESF Database Migration Toolkit y descargar la edición Professional del software.
    • Seguir las instrucciones de instalación proporcionadas por el asistente de instalación para instalar el software en tu sistema.

  • Abrir ESF Database Migration Toolkit:
    • Una vez instalado, abrir la aplicación ESF Database Migration Toolkit desde el menú de inicio o haciendo doble clic en el icono del programa en tu escritorio.

  • Conectar a la base de datos de origen (Microsoft Access):
    • En el panel izquierdo de la interfaz del software, hacer clic en "Source" para seleccionar la base de datos de origen.
    • Seleccionar "Microsoft Access" como el tipo de base de datos de origen.
    • Ingresar la información de conexión necesaria, como la ubicación del archivo de base de datos de Access, el nombre de usuario y la contraseña si es necesario.
    • Hacer clic en "Connect" para establecer la conexión con la base de datos de Access.

  • Conectar a la base de datos de destino (MySQL):
    • En el panel derecho de la interfaz del software, hacer clic en "Target" para seleccionar la base de datos de destino.
    • Seleccionar "MySQL" como el tipo de base de datos de destino.
    • Ingresar la información de conexión para la base de datos MySQL, como la dirección del servidor, el nombre de usuario, la contraseña y el nombre de la base de datos de destino.
    • Hacer clic en "Connect" para establecer la conexión con la base de datos MySQL.

  • Seleccionar las tablas para migrar:
    • En la sección de configuración de migración, seleccionar las tablas que deseas migrar de la base de datos de origen a la base de datos de destino.
    • Seleccionar todas las tablas o solo las que necesites migrar.

  • Configurar las opciones de migración:
    • Configurar las opciones de migración según tus necesidades. Estas opciones pueden incluir la configuración de la estructura de la tabla, la migración de datos, la creación de índices, la migración de claves primarias y externas, etc.

  • Iniciar el proceso de migración:
    • Una vez que estén configuradas todas las opciones de migración, hacer clic en el botón "Migrate" o "Start Migration" para iniciar el proceso de migración.
    • Durante el proceso de migración, ESF Database Migration Toolkit copiará la estructura de la tabla y los datos de la base de datos de origen a la base de datos de destino.

  • Verificar el resultado de la migración:
    • Una vez completada la migración, verificar el resultado para asegurar que todas las tablas y datos se hayan migrado correctamente.
    • Revisar los registros de migración y los mensajes de estado proporcionados por ESF Database Migration Toolkit para identificar cualquier problema o error durante el proceso de migración.

  • Realizar pruebas y ajustes si es necesario:
    • Después de la migración, realizar pruebas exhaustivas en la base de datos de destino para asegurarte de que todas las funcionalidades y datos se hayan transferido correctamente.
    • En caso de que exista algún problema o discrepancia, ajustar la configuración de migración según sea necesario y vuelve a ejecutar el proceso de migración.

  • Finalizar y guardar la configuración de migración:
    • Una vez que sea correcto el resultado de la migración, guardar la configuración de migración para futuras referencias o migraciones adicionales.
