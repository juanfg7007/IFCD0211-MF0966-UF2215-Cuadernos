UF2215. UD4. Actividad 1. Caso práctico: Entornos 4GL


Enunciado (Descripción)

Dentro de los lenguajes 4GL podemos distinguir el lenguaje de consulta de base de datos SQL.

A continuación vamos a aprender a realizar consultas en varias tablas de la base de datos al mismo tiempo usando el lenguaje SQL de Oracle.

Para ello usaremos los diferentes tipos de combinaciones externas (NATURAL LEFT OUTER JOIN, NATURAL RIGHT OUTER JOIN, NATURAL FULL OUTER JOIN) o de combinaciones internas (NATURAL JOIN, INNER JOIN).

Supongamos que nuestra base de datos contiene las siguientes tablas:

-- -----------------------------------------------------
-- Tabla Empresas
-- -----------------------------------------------------
CREATE TABLE Empresas(
Id_Empresa INT NOT NULL,
Nombre_EmpresaVARCHAR(40) NOT NULL,
PRIMARY KEY (Id_Empresa));
-- -----------------------------------------------------
-- TablaServicios
-- -----------------------------------------------------
CREATE TABLE Servicios (
Id INT,
Id_Empresa INT NOT NULL,
ServicioVARCHAR(40),
PRIMARY KEY (Id));

Estas tablas están rellenas con los siguientes valores:

Preguntas / Actividades a realizar 

Pregunta 1. Queremos obtener la combinación natural interna de Empresas y servicios para conocer los siguientes aspectos:

- Nombre de la Empresa
- Servicios de la Empresa

Pregunta 2. Aplica la combinación externa izquierda a las tablas Empresas y servicios. ¿Qué resultado obtenemos?

Pregunta 3. Aplica la combinación externa derecha a las tablas Empresas y servicios. ¿Qué resultado obtenemos?

Pregunta 4. Aplica la combinación externa plena a las tablas Empresas y servicios. ¿Cómo podemos hacer esta operación en Oracle?

Pregunta 5. ¿Qué ocurre en este ejemplo si usamos la combinación interna INNER JOIN?
