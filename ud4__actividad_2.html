Recordemos que el código SQL de la creación de estas tablas quedaba de la siguiente manera:

CREATE TABLE Alumnos (Cod_Alumno INT PRIMARY KEY,Nombre_Alumno VARCHAR(40) NOT NULL,NIF VARCHAR(10) NOT NULL,Direccion VARCHAR(40) NULL,Ciudad VARCHAR(25) NULL);
CREATE TABLE Asignaturas (Codigo_asig INT PRIMARY KEY,nombre_asig VARCHAR(40) NOT NULL,horas_lectivas INT NOT NULL,fecha_inicio DATE NOT NULL,
fecha_fin DATE NOT NULL,Codigo_Alumno INT NOT NULL,CONSTRAINT fk_Asignaturas_Alumnos FOREIGN KEY(Codigo_Alumno) REFERENCES Alumnos(Cod_Alumno)
ON DELETE CASCADE);

Preguntas / Actividades a realizar 

Sabiendo que los generadores de formularios, son herramientas útiles para agilizar el proceso de captura/presentación de la información del modelo de datos, enfatizándose en la velocidad y aumento de la productividad, nos centraremos en el diseño de formularios con la herramienta developer suite 10g de Oracle para ello:

Pregunta 1. Realiza la instalación de la herramienta Developer suite 10g.

Pregunta 2. Diseña un formulario básico,en el SGBD Oracle,para la tabla Alumnos, donde se muestren los campos Cod_Alumno, Nombre_Alumno y NIF ayudándote del asistente.

Pregunta 3. En este apartado vamos a hacer uso del asistente de formularios, de la herramienta de Developer Suite 10g de Oracle, llamada Reports Builder ,para generar un informe de la tabla Alumnos donde se muestren los campos Cod_Alumno, Nombre_Alumno y NIF.Explica los pasos a seguir para crear dicho informe.

Pregunta 4. Ayudándote de un programa de prueba llamado Altova DatabaseSpy 2015 genera un gráfico para nuestra base de datos Oracle XE haciendo uso de la tabla Asignaturas donde se muestren por medio de gráfico de barras las horas lectivas por asignaturas.
