UF2215. UD1. Actividad 4. Ejercicio teórico-práctico: Enterprise Monitor para administración de datos


Enunciado (Descripción)

Existen en cada SGBD herramientas avanzadas como en el caso de MySQL Enterprise monitor que facilitan la automatización y control de los servidores sin necesidad de recurrir a la programación.
A continuación nos centraremos en el funcionamiento de la aplicación Enterprise Monitor. Se trata de un programa que viene en la versión Enterprise de MySQL y que se puede conseguir registrándose en la página de Oracle MySQL mediante un año de suscripción.

Pregunta A

La arquitectura del MySQL Enterprise Monitor proporciona una solución completa y eficiente para la supervisión y gestión de servidores de bases de datos MySQL en entornos empresariales:

  • Servidor del Enterprise Monitor (EMS). Es el componente central que coordina y gestiona todas las funciones del sistema. El EMS actúa como el cerebro del sistema, recopilando datos, procesándolos y proporcionando la interfaz de usuario para la administración y supervisión.

  • Agentes en servidores MySQL. Cada servidor MySQL que se desea supervisar tiene un agente instalado. Estos agentes recopilan datos en tiempo real sobre el rendimiento, la disponibilidad y otros aspectos relevantes del servidor.

  • Base de Datos de almacenamiento. Se utiliza para almacenar datos recopilados por los agentes. Puede ser una base de datos dedicada para garantizar la eficiencia y la escalabilidad.

  • Consola web del Enterprise Monitor. Proporciona una interfaz de usuario gráfica y fácil de usar. Permite a los administradores visualizar datos en tiempo real, configurar alertas, acceder a informes y realizar acciones de administración.

  • Motor de análisis. Procesa datos recopilados, identifica patrones, tendencias y comportamientos anómalos. También realiza análisis predictivos.

  • Generación de informes. Produce informes detallados y personalizables sobre el rendimiento y la salud del servidor MySQL.

  • Sistema de alertas y acciones correctivas. Configura alertas para eventos críticos. Puede automatizar acciones correctivas, como ajustes de configuración, escalado automático o ejecución de scripts predefinidos.

  • Mecanismos de seguridad. Incorpora medidas de seguridad para garantizar la confidencialidad e integridad de los datos. Se integra con sistemas de autenticación y autorización existentes.

  • Sistema de actualización. Permite la aplicación sencilla de parches, actualizaciones y mejoras en el Enterprise Monitor para mantener la eficacia y la seguridad.

Pregunta B1

La visualización de los signos vitales de un servidor y el uso del Enterprise Dashboard en MySQL Enterprise Monitor permiten evaluar y gestionar el rendimiento de los servidores MySQL. Los pasos y conceptos relacionados con estas funcionalidades son:

  • Visualización de los signos vitales
    • Los "signos vitales" se refieren a indicadores clave del rendimiento y la salud del servidor MySQL. Estos pueden incluir métricas relacionadas con la CPU, RAM, caché, BIOS y otros aspectos críticos.
    • En la pestaña "Monitor" de la aplicación, se presenta un resumen de estos signos vitales, permitiendo a los administradores identificar posibles problemas o anomalías en el servidor.
    • Los "Critical Events" indican eventos importantes o advertencias que pueden requerir atención inmediata.

  • Enterprise Dashboard
    • El Enterprise Dashboard es una herramienta integral que ofrece una vista consolidada y detallada del estado y la actividad del servidor MySQL.
    • Proporciona información clave sobre el servidor seleccionado, como la versión del servidor, el tiempo de ejecución, el puerto de conexión, el socket utilizado, el sistema operativo, el espacio en disco y otros detalles relevantes.
    • Permite visualizar la actividad de la base de datos a través de gráficos interactivos y proporciona acceso a parámetros de configuración esenciales de MySQL.

  • Pasos para acceder y utilizar el Enterprise Dashboard
    • Inicio de sesión en la aplicación con las credenciales del usuario administrador.
    • Desde la lista de servidores (All Servers), seleccionar el servidor específico que desea analizar.
    • Acceder a la pestaña Monitor para obtener una visión general de los signos vitales y posibles eventos críticos.
    • En la pestaña Enterprise Dashboard para acceder a una vista completa del servidor, incluyendo detalles técnicos y métricas de rendimiento.

Pregunta B2

En la pestaña Monitor del MySQL Enterprise Monitor, específicamente en la sección de Graphs, se pueden visualizar varias métricas relacionadas con la actividad del servidor MySQL. Entre estas métricas se incluyen el número de conexiones activas y abortadas, así como las consultas SELECT, INSERT y UPDATE que se han ejecutado en cada momento. Estas métricas se presentan en forma de gráficos que muestran la actividad a lo largo del tiempo, lo que permite a los administradores monitorear de manera efectiva la carga del servidor y la actividad de las consultas.

Pregunta B3

El Analizador de Consultas (Query Analyzer) del MySQL Enterprise Monitor proporciona herramientas para identificar consultas que pueden beneficiarse de optimización. Una vez que se ha seleccionado un pico estimado en una fracción de tiempo específica en una gráfica de actividad, se puede abrir el Analizador de Consultas para examinar las consultas ejecutadas durante ese período. Al analizar cada consulta, el Analizador de Consultas proporciona información valiosa sobre el tiempo de ejecución de la consulta, el uso de índices, el usuario que la ejecutó y otras métricas relacionadas con el rendimiento. Las consultas que tienen un tiempo de ejecución prolongado o que no hacen un uso eficiente de los índices son candidatas a optimización. Además, las consultas que se ejecutan con frecuencia y contribuyen significativamente a la carga del servidor también pueden ser prioritarias para la optimización.

Pregunta B4

El monitor utiliza las reglas advisors para comparar las prácticas recomendadas con la configuración actual del servidor MySQL. Estas reglas advisors están diseñadas para identificar áreas donde la configuración del servidor puede no estar optimizada o donde se pueden aplicar ajustes para mejorar el rendimiento y la estabilidad del servidor. Cuando se habilitan las reglas advisors y se produce un evento que viola una de estas reglas, el monitor genera una alerta en la pestaña Events. Por ejemplo, si se detecta que el caché de consultas no está habilitado y esta es una práctica recomendada, el monitor generará una alerta con recomendaciones para habilitar el caché de consultas y mejorar el rendimiento del servidor. Los administradores pueden luego revisar estas alertas, aplicar las recomendaciones sugeridas y documentar las acciones tomadas en la pestaña Close Event para mantener un registro de los cambios realizados en la configuración del servidor.
