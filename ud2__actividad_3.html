UF2215. UD2. Actividad 3. Ejercicio teórico-práctico: ETL


Enunciados (Descripción)

La Universidad requiere realizar un proceso a una tabla persona_bd (que forma parte del ERP, es decir, del sistema de planificación de recursos donde se integran todas las actividades del centro formativo) junto con la información contenida en un archivo de Excel (Sistema de Información Académica) para almacenarla en una DW (Data Warehouse o almacén de datos que es una colección de datos orientada al ámbito académico, integrado, no volátil y variable en el tiempo, que ayuda a la toma de decisiones) llamada dim_persona. Debido a que los valores del campo tarjeta de identidad del archivo de Excel no son iguales a los que están contenidos en el campo tipo de identificación de la tabla persona_bd nos surgen las siguientes cuestiones:


Preguntas / Actividades a realizar 

Pregunta 1A. ¿Qué tipo de proceso se debe aplicar en este caso práctico planteado?

Pregunta 1B. Describe las distintas fases de este proceso para este caso.

UD2 - Actividad 3 - Resolución

Pregunta 1A

En este caso práctico, se debe aplicar un proceso de ETL (Extract, Transform, Load). El ETL es un conjunto de prácticas y herramientas utilizado en el campo de la minería de datos y los almacenes de datos para la integración y consolidación de datos desde múltiples fuentes.

En este escenario, la Universidad busca combinar la información de la tabla persona_bd del ERP con los datos de un archivo de Excel del Sistema de Información Académica para almacenarla en un Data Warehouse llamada dim_persona.

Pregunta 1B

El proceso ETL consta de tres fases fundamentales:

  • Extract (Extracción)
    • En esta fase, se extraen datos de las fuentes de origen. En el caso planteado, se extraerán datos de la tabla persona_bd del ERP y del archivo de Excel del Sistema de Información Académica.
    • Para abordar la discrepancia entre los valores del campo `tarjeta de identidad` del archivo Excel y el campo tipo de identificación de la tabla persona_bd, es necesario realizar una extracción cuidadosa, identificando y manejando las diferencias de formato o codificación entre los datos.

  • Transform (Transformación)
    • Los datos extraídos son transformados en esta fase para asegurar que cumplan con los requisitos del Data Warehouse y puedan ser integrados correctamente.
    • En el caso de la discrepancia en los valores del campo de identificación, se llevarán a cabo transformaciones para alinearlos. Esto podría incluir la normalización de formatos, la corrección de errores tipográficos y la aplicación de reglas de conversión.
    • Además, se pueden realizar otras transformaciones, como la conversión de tipos de datos, la creación de campos calculados y la limpieza de datos.

  • Load (Carga)
    • En la fase final, los datos transformados se cargan en el Data Warehouse (dim_persona en este caso).
    • La carga implica colocar los datos en las estructuras definidas en el Data Warehouse para su posterior análisis y consulta.
    • Durante este proceso, se debe garantizar la integridad referencial y la consistencia de los datos cargados.
