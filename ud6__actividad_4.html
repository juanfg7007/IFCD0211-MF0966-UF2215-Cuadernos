UF2215. UD6. Actividad 4. Ejercicio teórico-práctico: Métodos de comunicación de un SGBD


Preguntas / Actividades a realizar 

Pregunta 1. Enunciar las características generales de los medios de acceso más comunes a los métodos o protocolos de comunicación de los que dispone un SGBD para conectarse con un servidor.

UD6 - Actividad 4 - Resolución

Pulsa aquí para visualizar: UD6 - Actividad 4 - Resolución

Los medios de acceso a los métodos o protocolos de comunicación de un Sistema Gestor de Bases de Datos (SGBD) facilitan el establecimiento y gestión de la conexión entre el cliente y el servidor de la base de datos. Estos medios de acceso permiten que las aplicaciones y los usuarios interactúen con la base de datos de manera eficiente y segura. Las características generales de los medios de acceso más comunes son:

  • ODBC (Open Database Connectivity)
    • ODBC es una interfaz estándar de acceso a datos diseñada para permitir que las aplicaciones accedan a cualquier base de datos utilizando SQL como lenguaje de acceso.
    • Permite la conexión a una amplia variedad de bases de datos, independientemente del proveedor.
    • Proporciona un conjunto de funciones y API para establecer la conexión, enviar consultas SQL y recuperar resultados.
    • Es compatible con múltiples plataformas y sistemas operativos.

  • JDBC (Java Database Connectivity)
    • JDBC es una API de Java que permite que las aplicaciones escritas en Java accedan y manipulen datos almacenados en una base de datos relacional.
    • Proporciona clases e interfaces para interactuar con el SGBD, incluida la conexión a la base de datos, la ejecución de consultas SQL y la gestión de transacciones.
    • Facilita la portabilidad de las aplicaciones Java, ya que el código JDBC es independiente del SGBD subyacente.
    • Permite la programación de bases de datos desde cualquier entorno que admita Java.

  • OLE DB (Object Linking and Embedding, Database)
    • OLE DB es una API de Microsoft que permite el acceso a datos de diversas fuentes mediante interfaces de objetos COM (Component Object Model).
    • Ofrece una interfaz unificada para acceder a una amplia variedad de fuentes de datos, incluidas bases de datos relacionales, archivos de texto, hojas de cálculo, entre otros.
    • Permite el acceso a datos de manera eficiente y flexible utilizando tecnologías COM.
    • Es compatible con aplicaciones desarrolladas en entornos de desarrollo de Microsoft, como Visual Basic y C++.

  • ADO.NET (ActiveX Data Objects .NET)
    • ADO.NET es una tecnología de acceso a datos de Microsoft diseñada específicamente para trabajar con el entorno de desarrollo .NET.
    • Proporciona clases y componentes para acceder a datos de manera eficiente y administrar conexiones, comandos y conjuntos de resultados.
    • Permite el acceso a una amplia gama de fuentes de datos, incluidas bases de datos relacionales, XML y servicios web.
    • Facilita el desarrollo de aplicaciones web y de escritorio en el entorno .NET Framework.
